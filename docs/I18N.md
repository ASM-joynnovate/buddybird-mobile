# I18N — 로케일·번역·로케일 콘텐츠 정책

앱의 언어 결정과 로케일별 콘텐츠(프리셋 단어·오디오) 규칙. 이관 이력·감사 기록은 `docs/I18N-COMPLETION.md`(BB-155) 참고.

## 1. 로케일 결정

- 지원 로케일은 `SUPPORTED_LOCALES = ['ko', 'en']` (`features/i18n/i18n-resources.ts`) — 새 로케일 추가는 별도 정책 결정 필요
- 앱 로케일은 **기기 언어를 따른다** (`expo-localization` → `normalizeLocale`, 앱 시작 시 1회 결정). 인앱 언어 선택 UI는 만들지 않는다
- **미지원 언어 기기는 `DEFAULT_LOCALE = 'en'`으로 fallback** — UI와 프리셋 단어가 함께 영어를 따른다 (2026-07-16 ko→en 전환, ko 기기는 기존대로 한국어)

## 2. 번역 리소스

- 모든 사용자 노출 문자열(화면 텍스트·Alert·접근성 라벨)은 `features/i18n/i18n-resources.ts` 경유 — 하드코딩 금지. `AppCopy` 타입에 키를 먼저 추가하면 ko/en 동시 작성이 컴파일 타임에 강제된다
- 접근성 전용 키는 `*A11y` 접미사, ko 카피는 해요체, en 카피는 typographic apostrophe(`’`) 사용
- features 레이어는 JSX/훅 금지이므로 `t`(또는 `locale`)를 마지막 인자로 주입받는다 — 선례: `profile-validation.ts`, `duration-format.ts`, `getSpeciesOptions(locale)`
- 로그 전용 메시지(`reportError`/`console.warn`에만 전달)와 `components/debug/*`는 번역 비대상

## 3. 로케일 콘텐츠 (프리셋 단어·오디오)

- 프리셋 단어는 UI 카피가 아니라 **앵무새 학습 콘텐츠** — 로케일별 시드로 분기한다 (`SEED_PRESETS_BY_LOCALE`, `features/word-library/word-library-model.ts`)
  - ko: 안녕/사과/사랑해/다녀와 · **en: 시드 없음** (제공할 영어 프리셋 음원 없음 — 빈 목록 empty state 노출)
  - 시드는 빈 store hydrate 시 1회, 시드가 0개면 빈 store 를 저장하지 않는다 — 기존 설치에 소급하지 않는다
- 프리셋 오디오는 `assets/audio/<locale-dir>/` 로케일 디렉토리로 관리 (현재 `ko-kr/`만), 파일명은 `default_<단어>.m4a` 양식
- `preset://` 오디오 매핑 키(`PRESET_AUDIO_MODULES`, `features/audio/audio-source-resolver.ts`)는 로케일 무관 단일 네임스페이스 — 향후 비-ko 프리셋 키는 로케일 접두 필수 (예: `en-hello`), ko 키는 하위 호환을 위해 무접두 유지

## 4. 네이티브 문자열

- Android 네이티브(알림 등) 문자열은 JS i18n이 아니라 `res/values/strings.xml`(ko) + `res/values-en/strings.xml`로 관리 — OS 로케일을 따른다

## 5. 검증 grep

```bash
# 하드코딩 한국어 잔재 (행 시작 주석 제외 — under-filter 안전망, 잔여는 육안 분류)
grep -rn "[가-힣]" app components features hooks constants \
  | grep -v "features/i18n/i18n-resources.ts" | grep -v "components/debug/" \
  | grep -vE ":[0-9]+:\s*(//|/?\*)"

# 네이티브 코드 한국어 (리소스 xml 제외)
grep -rn "[가-힣]" modules --include="*.kt" --include="*.swift"

# 프리셋 시드 키 ↔ 오디오 맵 키 일치 (수동 대조)
grep -n "key: '" features/word-library/word-library-model.ts
grep -n "require(" features/audio/audio-source-resolver.ts
```
